Owner guide

Editing the live site.

The redesigned site uses Markdown content, generated static pages and small JSON data files. This keeps it free-hosting friendly while making edits safer than one giant posts file.

Create a new review

Copy an existing file from content/reviews/, rename it with a clean slug, update the YAML frontmatter, then write the article below the frontmatter using Markdown headings. Mark the status honestly: Owned, Handled, Research brief or Wishlist.

Add referral links

Add product links inside the article frontmatter under products, or add reusable items to data/products.json. Use full affiliate links only after joining the programme. Keep rel="sponsored nofollow noopener" on product links.

Edit the Kit Builder database

Update data/kit-items.json to add knives, stones, strops, boards, storage or utensils. Keep attributes consistent: category, profile, edgeLengthMm, steelType, handleType, bestFor, maintenance, tags and learnUrl.

Edit the Knife Finder

Update data/finder.json. Each rule has a match, recommendation, caveat, maintenance kit and article IDs. This is where paths like low-fuss vegetable prep point to a VG10 stainless-clad nakiri and the right board/stone/strop setup.

Publish with Netlify

Push changes to GitHub. Netlify runs python3 scripts/build.py, regenerates static pages, and publishes the folder. Decap CMS in /admin/ can edit Markdown once Git Gateway is configured.

Before publishing

Run python3 scripts/build.py, then python3 -m http.server 8080. Check the homepage, article page, recommendations page and disclosure page before deploying.
